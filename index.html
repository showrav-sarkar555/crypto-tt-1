Lecture 1

Security Foundations

Why computer security matters and the fundamental properties every secure system must uphold.

📌 What is Computer Security?

  • Protection of computer systems operating in adversarial environments with possible adversaries.
  • Aims to allow intended use and prevent unintended use.
  • Focus: why systems are insecure and how to make them secure — not how to break in.

🏗️ Security Properties

🔒

Confidentiality

Keeping information secret from all but those who are authorized to see it.

✅

Integrity

Ensuring information has not been altered by unauthorized or unknown means.

🟢

Availability

Data/information is available when required by authorized users.

🪪

Authenticity

Data/information origin is identifiable accurately; entity identity is verifiable.

👤

Anonymity

Actions or data not relatable to a particular individual.

🌐 Motivations — Why This Matters

  • ~8.8 billion mobiles, ~6 billion Internet users, ~378 million domain names worldwide.
  • ~$6,420 billion in retail e-commerce sales and growing.
  • Almost all spheres of life depend on the Internet — trend will continue to grow.
  • Real incidents: Mirai Botnet DDoS, Heartbleed, WannaCry ransomware.
Ethical Reminder: "With great power comes great responsibility!" — Course knowledge should never be used for malicious cracking. If you discover a security hole, contact the vendor; experiment only on your own machines/networks.
Lecture 2

Security Goals & Attacks

Deep dive into the CIA Triad, additional properties, and the taxonomy of security attacks.

📖 Common Criteria (CC) Terminology

CC is an international set of guidelines and specifications by ISO for evaluating information security products, specifically to ensure they meet an agreed-upon security standard for government deployments.

  • Asset: Something valuable to its owner.
  • Vulnerability: A weakness in the asset.
  • Threat: Exploits vulnerabilities within an asset to violate its security; increases the risk of abuse.
  • Threat Agent (Attacker/Adversary): Values assets and wants to abuse them.
  • Countermeasure: Adopted by owners to minimize risks.

Relationships: Threat agents exploit vulnerabilities → threats increase risk → owners adopt countermeasures.

🔍 Defining Security

  • The security of a system, application, or protocol is always relative to:
    • A set of desired properties
    • An adversary (attacker) with specific capabilities
  • Academic study of security is NOT about breaking into systems or launching attacks.
  • Focus: exploring why a system is insecure and how to make them secure.

🔺 The CIA Triad — In Detail

🔒 Confidentiality (Secrecy & Privacy)

  • Secrecy: Protecting unauthorized information access and disclosure.
  • Privacy: Protecting personal/proprietary information — individuals control what info is collected, stored, and disclosed.
  • Historical: Julius Caesar used a simple cipher to communicate with generals.

Tools of Confidentiality:

  • Encryption: Transform info using a secret key; can only be read with the decryption key.
  • Access Control: Rules/policies limiting access to those with "need to know" — by identity or role.
  • Authentication: Determining identity via something you have (smart card), know (password), or are (biometric).
  • Authorization: Determining if a person/system is allowed access based on policy.
  • Physical Security: Locks, windowless rooms, Faraday cages, etc.

✅ Integrity

  • Data Integrity: Info/programs changed only in specified, authorized manner.
  • System Integrity: System performs its intended function, free from unauthorized manipulation.

Tools of Integrity:

  • Backups: Periodic archiving of data.
  • Checksums: Map file contents to a numerical value; even a single bit flip changes the output.
  • Data Correcting Codes: Store data so small changes are detected and automatically corrected.

🟢 Availability

  • Ensuring timely, reliable access to information or systems.
  • Loss can be from accidental events (fire, disk failure) or malicious attacks (DDoS).

Tools: Physical protections, computational redundancies (fallback servers).

➕ Additional Security Properties

Property Description
Authenticity Entity/data origin is genuine, verifiable, and trusted. Source authenticity + Entity authentication.
Accountability / Non-Repudiation Actions traceable uniquely to an entity; a party cannot deny a certain action.
Anonymity Certain records or transactions not attributable to any individual.

🪪 Authenticity — In Detail

  • The property of an entity or data source being genuine, verifiable, and trusted — confidence in the validity of a transmission, message, or message originator.
  • Verifying that users are who they say they are and that each input arrived from a trusted source.
  • Example: Purporting someone else's identity for malicious purposes.
  • Data Authentication: Deals with source authenticity. Mechanism: Digital Signature.
  • Entity Authentication: Deals with identity authenticity. For a person:
    • Something you have — e.g., ID cards, smart cards
    • Something you know — e.g., a password or secret key
    • Something you are — e.g., biometric (fingerprint, face, iris scanning)

📋 Accountability / Non-Repudiation — In Detail

  • Ensures the actions of an entity can be traced uniquely to that entity.
  • True secure systems are not yet an achievable goal — so we must be able to trace a security breach to a responsible party.
  • This supports:
    • Non-repudiation — a party cannot deny a certain action
    • Fault isolation
    • After-action recovery and legal action
  • Mechanism: Via a secure audit trail.
  • Challenge: Creating an audit trail with machine logs is tricky — if a system is compromised, logs may also be tampered with. Solution approach: send log messages to an append-only file.

👤 Anonymity — In Detail

  • The property that certain records or transactions are not attributable to any individual.
  • Tools of Anonymity:
    • Mixing: Intertwining transactions, information, or communications in a way that cannot be traced to any individual.
    • Proxies: Trusted agents willing to engage in actions for an individual in a way that cannot be traced back to that person.
    • Pseudonyms: Fictional identities that can fill in for real identities in communications and transactions, but are otherwise known only to a trusted entity.

⚔️ Security Attacks

Passive Attacks

Eavesdropping on or monitoring transmissions to obtain and analyze information.

  • Eavesdropping: Intercepting information during transmission.
  • Traffic Analysis: Observing patterns of communication.

Active Attacks

Modification of data stream or creation of a false stream.

  • Masquerade: Fabricating information appearing to be from another source.
  • Modification (Alteration): Unauthorized modification of information.
  • Replay: Observe, analyze, then replay captured data.
  • DoS: Interruption/degradation of data service (e.g., email spam filling mail queue).
Lecture 4

Cryptosystem Basics & Brute-Force

Formal encryption models, symmetric vs. public-key crypto, and brute-force attacks.

🔐 Encryption Fundamentals

  • Goal: Allow Alice & Bob to communicate confidentially over an insecure channel subject to eavesdropping.
  • Plaintext (M) → Encryption Algorithm E → Ciphertext (C).
  • Equations: C = E(M) and M = D(C).
  • Encryption/decryption algorithms are public; only the key is secret.
  • If the decryption key can be derived from the encryption key → encryption key must also be kept secret.

📋 Seven Components of a Cryptosystem

  1. Set of possible plaintexts
  2. Set of possible ciphertexts
  3. Set of encryption keys
  4. Set of decryption keys
  5. Correspondence between encryption and decryption keys
  6. The encryption algorithm
  7. The decryption algorithm

🏛️ Caesar Cipher

  • Replace each letter with the one "three over" in the alphabet — a shift cipher.
  • Formula: s(c, k) where c = letter, k = key. If k > 0 → forward shift (encrypt); k < 0 → backward (decrypt).
  • Example: s(A, 3) = D, s(D, -3) = A.
  • Uses modulo operation for circular wrap-around.

🔑 Symmetric vs. Public-Key Cryptography

Symmetric Key

  • Fast and efficient
  • Main problem: key distribution
  • Requires n(n-1)/2 keys for n parties
  • Example: AES (256-bit keys)

Public Key

  • Only one key pair per recipient
  • Much slower than symmetric
  • Larger key lengths (RSA: 2048-bit)
  • Often used just to exchange a shared secret key, then switch to symmetric

🧹 Brute-Force Attack

  • Try different probable keys over ciphertext to decrypt it to meaningful text.
  • Encryption/decryption functions are open; only the key is secret.
  • If plaintext is arbitrary binary → attack cannot succeed (no way to distinguish valid message).
  • If plaintext is natural language → adversary hopes only a small subset of decryption results will be meaningful.
  • Example: decrypting to "NGGNPXNGQNJABAVEIVARORNPU" → dismiss! Decrypting to "ATTACKATDAWNONIRVINEBEACH" → found it!
  • Key should be a sufficiently long random value to make exhaustive search infeasible.
Unicity Connection: For English + 256-bit AES, the unicity distance is about 304 bits (38 ASCII characters) — only half a line of text! Brute-force is likely to succeed for natural-language messages that are not too short.

🕵️ Man-In-The-Middle (MITM) Attack

  • Charlie sits between Alice and Bob; can view, change, add, or delete traffic.
  • Charlie could be: ISP, VPN provider, WiFi provider, an attacker re-routing connections, or even an incompetent admin.
  • Straightforward crypto ensures confidentiality but not authenticity/integrity if adversary can modify ciphertext.
  • Adversary can alter ciphertext/signature but cannot choose or figure out resulting plaintext (no decryption key).
  • Effective only when any arbitrary bit sequence is a possible message (e.g., transmitting a random symmetric key).
MITM Signature Attack Flow:
Bob sends (M, S) → encrypted as (C, S) → Eve intercepts → modifies to (C', S') → Alice receives (C', S')
• M' will be different from original message M
• When Alice verifies digital signature S', she obtains M' by decrypting S'
• Alice is led to believe that Bob signed M' instead of M
• The adversary can arbitrarily alter the transmitted ciphertext or signature, but cannot choose or figure out the resulting plaintext

✍️ Digital Signature

  • A mechanism that allows a sender to attach a code (signature) to a message that acts as proof of origin and integrity.
  • Provides authenticity — the receiver can verify that the message truly came from the claimed sender.
  • Provides non-repudiation — the sender cannot deny having sent the message.
  • Based on public-key cryptography: sender signs with private key, receiver verifies with sender's public key.
Lecture 6

Cryptanalysis, Block Ciphers & Modes

Crypto primitives, attack classification, classic ciphers, block cipher structure, padding, and modes of operation.

🔑 Crypto Primitives

  • Alice wants to send a message (plaintext P) to Bob.
  • The communication channel is insecure and can be eavesdropped by an attacker Eve.
  • The goal is to allow secure exchanges of messages between Alice and Bob in an insecure channel.
  • Solution: Cryptography to the rescue! — transform plaintext into ciphertext so Eve cannot read it.

🔍 Cryptosystem Attacks (Cryptanalysis)

We assume the cryptanalyst knows the algorithms — only the keys are secret (open design principle). Security by obscurity fails because info can leak (stolen docs, bribed programmers, reverse engineering).

Attack Type What Cryptanalyst Has Goal
Ciphertext-Only Ciphertext(s) encrypted with same key K Determine plaintext or discover K
Known-Plaintext Plaintext-ciphertext pairs (same key K) Determine the key K
Chosen-Plaintext Can choose plaintexts and get ciphertexts (same K) Infer key behavior. Variants: offline (choose all in advance) and adaptive (iterative, based on previous results)
Chosen-Ciphertext Can choose ciphertexts and get plaintexts (same K) Same as chosen-plaintext but reversed; also has offline and adaptive variants

🔤 Classic Ciphers

Substitution Cipher

  • Each letter is uniquely replaced by another.
  • Caesar cipher is an example of a substitution cipher utilised by Julius Caesar:
    • Each letter in the plaintext is shifted three letters to the right.
    • When it reaches the end, it is wrapped back at the beginning.
    • Decryption requires a three left shift.

Substitution Cipher Attack (Frequency Analysis)

  • Even with a huge key space (26! ≈ 4×10²⁶ possible keys), a substitution cipher can be easily broken.
  • This is because letters in a natural language like English are not uniformly distributed.
  • Knowledge of letter frequencies (including pairs and triples) can be used in cryptanalytic attacks.
  • In English text: letter "E" occurs ~12% of the time, "T" occurs <10% of the time.
  • The most frequently occurring character in ciphertext from English substitution cipher probably corresponds to the letter "E", and so on.

Polygraphic Substitution Cipher

  • Encrypt digraphs (letter pairs) instead of single letters.
  • One easy way to express a digram substitution is using a two-dimensional table.
  • First letter in a pair specifies the row, second letter specifies the column; each entry is the unique two-letter substitution (called a S-box — substitution box).
  • This approach can be extended for binary values.

Vigenère Cipher

  • A polyalphabetic substitution cipher — uses a repeating keyword to vary the shift.
  • Unfortunately, it can be easily broken using statistical techniques (Kasiski examination, Index of Coincidence).

One-Time Pads

  • Provide perfect security, but have practical weaknesses:
  • The key has to be as long as the plaintext.
  • Keys can never be reused.
  • Repeated use of one-time pads allowed the U.S. to break some of the communications of Soviet spies during the Cold War.

Hill Cipher

  • A polygraphic substitution cipher based on matrix multiplication (linear algebra).
  • Uses an n×n key matrix; plaintext is broken into vectors of size n.

🧱 Block Cipher

  • Encrypts fixed-size blocks using the same secret key.
  • Requires plaintext length to be a multiple of block size.

PKCS#5 Padding

If plaintext length = n bytes and block size = b bytes:

Padding needed: k = b − (n mod b) bytes

Each padding byte has value k.

If n is already a multiple of b → add a full block of padding (to remove ambiguity).

Example: Block size = 128 bits (16 bytes), Plaintext = "Roberto" (7 bytes).
k = 16 − 7 = 9 → Pad with 9 bytes each valued 0x09.
Padded: "Roberto" + 09 09 09 09 09 09 09 09 09 = 16 bytes.

⚙️ Modes of Operation

ECB Electronic Codebook

Each block encrypted independently with same key.

✅ Simple, parallelizable, good for random data.

❌ Patterns in plaintext leak into ciphertext (penguin problem).

CBC Cipher-Block Chaining

Each block XORed with previous ciphertext block before encryption; uses an IV.

✅ Hides patterns. Most common mode. Fast & simple.

❌ Sequential dependency. Not suitable for packet-loss scenarios (streaming).

CFB Cipher Feedback

Stream-like feedback mode using ciphertext as feedback input.

✅ Useful for streaming data.

❌ Error in one ciphertext block propagates to next.

OFB Output Feedback

Keystream mode — generates keystream independent of plaintext/ciphertext.

✅ No error propagation from ciphertext corruption.

❌ Must never reuse IV with same key.

Playfair / Hill / Vigenère

Classical Cipher Techniques

Polyalphabetic and polygraphic cipher systems that improve upon simple substitution.

🔑 Vigenère Cipher

  • A polyalphabetic substitution cipher — uses a keyword to vary the shift for each letter.
  • Key is repeated to match the plaintext length.
  • Encryption: Cᵢ = (Pᵢ + Kᵢ) mod 26
  • Decryption: Pᵢ = (Cᵢ − Kᵢ) mod 26
  • Stronger than Caesar because each letter can shift by a different amount, defeating simple frequency analysis.
  • However, can still be broken using statistical techniques (Kasiski examination, Index of Coincidence).
Example: Encrypt "HELLO" with key "KEY"
Key stream: K E Y K E
H+K=R, E+E=I, L+Y=J, L+K=V, O+E=S → Ciphertext: RIJVS

📐 Playfair Cipher

  • Encrypts digraphs (pairs of letters) instead of single letters.
  • Uses a 5×5 key matrix (I and J share a cell).

Building the Matrix

  1. Write the keyword (removing duplicate letters).
  2. Fill remaining cells with unused letters of the alphabet (I/J combined).

Encryption Rules

  1. Split plaintext into digraphs. If a pair has two same letters, insert 'X' between them. If odd length, pad with 'X'.
  2. Same Row: Replace each letter with the one to its right (wrap around).
  3. Same Column: Replace each letter with the one below (wrap around).
  4. Rectangle: Each letter takes the letter in its own row but the other letter's column.
Why stronger than simple substitution? Encrypting pairs obscures single-letter frequency statistics. There are 26² = 676 possible digraphs vs. only 26 single letters, making frequency analysis much harder.

🧮 Hill Cipher

  • A polygraphic substitution cipher based on linear algebra.
  • Uses an n×n key matrix. Plaintext is broken into vectors of size n.
  • Encryption: C = K · P mod 26 (matrix multiplication).
  • Decryption: P = K⁻¹ · C mod 26 (requires modular inverse of key matrix).
  • Stronger diffusion — each ciphertext letter depends on multiple plaintext letters.
Unicity Distance

Unicity Distance in Cryptography

How much ciphertext is needed to uniquely determine the correct key?

📏 Definition

Unicity Distance is the minimum amount of ciphertext required to uniquely determine the correct encryption key.

In simple words: It tells us how much encrypted data is needed so that only ONE key can correctly decrypt it.

💡 The Core Idea

Short Ciphertext

Many keys may produce meaningful outputs → Cannot decide which key is correct.

Short ciphertext = Confusion

Long Ciphertext

Only one key will give a meaningful message → Can decide the correct key.

Long ciphertext = Clarity

📊 Example 1: Ciphertext < Unicity Distance

3 possible keys: Key 1 → Shift +1, Key 2 → Shift +2, Key 3 → Shift +3

Ciphertext: D

Trying all keys:

  • Key 1 → C (valid letter)
  • Key 2 → B (valid letter)
  • Key 3 → A (valid letter)

All results are valid! Cannot determine the correct key.

📊 Example 2: Ciphertext ≥ Unicity Distance

Ciphertext: KHOOR

Trying all keys:

  • Key 1 → JGNNQ (no meaning)
  • Key 2 → IFMMP (no meaning)
  • Key 3 → HELLO ✅ (meaningful!)

Only one key gives a meaningful result → Unique solution found!

Key Fact: For English + 256-bit AES, the unicity distance is about 304 bits (38 ASCII characters) — only half a line of text. This means brute-force decryption is likely to succeed for natural-language messages that are not too short.

Sample Questions — Part A

Detailed answers for all 11 theoretical questions.

Q1 Define system integrity in computer security. Explain how integrity differs from confidentiality with an example.

System Integrity

System integrity assures that a system performs its intended function in an unimpaired manner, free from deliberate or inadvertent unauthorized manipulation of the system.

Integrity vs. Confidentiality

Aspect Confidentiality Integrity
Definition Preventing unauthorized reading/disclosure of information Preventing unauthorized modification/alteration of information
Focus Who can see the data Who can change the data
Example: Consider an employee salary database:
• If salary data is leaked to unauthorized people → Confidentiality failure.
• If salary numbers are altered without authorization → Integrity failure.
Q2 Explain the CIA Triad with a real-world banking system example showing how each component is protected.

The CIA Triad is the foundational model for information security:

Component Banking Example
🔒 Confidentiality Customer account data, balances, and transaction details are encrypted (both at rest and in transit) and protected by strict access controls. Only authorized personnel with proper authentication can view sensitive records.
✅ Integrity Transactions are protected by checksums, digital signatures, and audit logs. Any unauthorized modification of account balances or transfer records would be detected. Double-entry accounting ensures consistency.
🟢 Availability Redundant servers, failover systems, and DDoS protection ensure banking services (ATMs, online banking, mobile apps) remain accessible 24/7. Disaster recovery plans maintain uptime during outages.
Q3 Describe different types of cryptanalysis attacks. Briefly explain ciphertext-only and chosen-plaintext attacks.

Cryptanalysis is the science of attacking cryptosystems. We assume the analyst knows the algorithms (open design principle) — only the keys are secret.

Types of Attacks:

  1. Ciphertext-Only Attack: The attacker has access to one or more ciphertexts (all encrypted with the same key K). Goal: determine the plaintext or discover K. This is the weakest attack model — the attacker has minimal information.
  2. Known-Plaintext Attack: Attacker has one or more plaintext-ciphertext pairs. Goal: determine key K.
  3. Chosen-Plaintext Attack: The attacker can choose plaintext messages and observe the corresponding ciphertexts (same key K). This allows the analyst to strategically probe the cipher.
    • Offline: All plaintexts chosen in advance.
    • Adaptive: Plaintexts chosen iteratively based on previous results.
  4. Chosen-Ciphertext Attack: Attacker can choose ciphertexts and obtain corresponding plaintexts. Also has offline and adaptive variants.
Q4 What is a brute force attack? Under what conditions does brute force become practical?

Brute Force Attack

An exhaustive key search — the attacker tries every possible key to decrypt ciphertext until meaningful plaintext appears. The encryption and decryption algorithms are public; only the key is secret.

When Brute Force Becomes Practical:

  1. Small keyspace: If the number of possible keys is small enough to enumerate.
  2. Weak/predictable keys: When keys are not truly random (e.g., weak passwords).
  3. High computational resources: Attacker has access to powerful hardware (GPUs, ASICs, botnets).
  4. Sufficient ciphertext available: Enough ciphertext that only one valid plaintext exists (related to unicity distance). For natural language, the attacker can recognize meaningful text and confirm the correct key.
If the plaintext is an arbitrary binary string, brute force cannot succeed — there's no way to distinguish a valid message from random garbage.
Q5 Explain the concept of a block cipher. Discuss different types of block cipher modes of operation (ECB, CBC, CFB, OFB). Mention one advantage and one disadvantage of any one mode.

Block Cipher

A block cipher encrypts fixed-size blocks of plaintext (e.g., 64 or 128 bits) using the same secret key. The plaintext must be padded so its length is a multiple of the block size.

Modes of Operation

Mode How It Works Key Feature
ECB Each block encrypted independently with same key Simplest mode; parallelizable
CBC Each plaintext block XORed with previous ciphertext block; uses IV Hides patterns; most common
CFB Previous ciphertext fed back as input; stream-like mode Useful for streaming data
OFB Generates keystream from IV; XORed with plaintext No error propagation
CBC — Advantage & Disadvantage:
✅ Advantage: Hides plaintext patterns — identical plaintext blocks produce different ciphertext blocks (due to chaining).
❌ Disadvantage: Requires sequential processing (no parallelization for encryption). Not suitable for packet-loss applications (music/video streaming) because errors propagate.
Q6 What is the PKCS#5 padding method? Why is padding necessary in block cipher encryption?

PKCS#5 Padding Method

When plaintext length is not a multiple of the block size, PKCS#5 appends k bytes each with value k, where:

k = b − (n mod b)
where b = block size in bytes, n = plaintext length in bytes.

If n is already a multiple of b, a full block of padding is added (k = b), where each byte has value b.

Why Padding is Necessary

  1. Block ciphers require that the plaintext length be an exact multiple of the block size. The last block must be filled.
  2. Padding must be unambiguous — you can't just add zeros, because the receiver wouldn't know how many padding bytes to strip off.
  3. PKCS#5's design makes unpadding deterministic: read the last byte's value → that tells you exactly how many padding bytes to remove.
Q7 Explain how the Vigenère cipher improves security over the Caesar cipher.
Feature Caesar Cipher Vigenère Cipher
Type Monoalphabetic substitution Polyalphabetic substitution
Key Single fixed shift value Keyword (multiple shifts)
How it works All letters shifted by same amount Each letter shifted by a different amount based on the key
Frequency analysis Easily broken — letter frequency preserved Much harder — same letter can encrypt to different ciphers
Keyspace Only 26 possible keys 26ⁿ keys for keyword length n

The fundamental improvement is that Vigenère uses varying shifts from a repeating keyword, which means the same plaintext letter can produce different ciphertext letters at different positions. This defeats simple frequency analysis that easily cracks Caesar.

Q8 Describe the Playfair cipher encryption technique. Why is it considered stronger than simple substitution ciphers?

Playfair Encryption Technique

  1. Build a 5×5 Key Matrix: Write the keyword (removing duplicates), then fill remaining cells with unused alphabet letters. I and J share one cell.
  2. Prepare Plaintext: Split into digraphs (2-letter pairs). If a pair has identical letters, insert 'X' between them. If odd length, pad with 'X'.
  3. Encrypt Each Digraph:
    • Same Row: Replace each letter with the one to its right (wrap around).
    • Same Column: Replace each letter with the one below (wrap around).
    • Rectangle: Each letter replaced by the letter in its own row but the other letter's column.

Why Stronger Than Simple Substitution?

  • Encrypts letter pairs, not individual letters — obscures single-letter frequency statistics.
  • There are 676 possible digraphs (26×26) vs. only 26 single letters, making frequency analysis significantly harder.
  • The same letter can encrypt to different ciphertext letters depending on its pair.
Q9 An attacker intercepts encrypted messages and tries every possible key until meaningful text appears. Identify the attack. Which security principle is being challenged?

Attack: This is a Brute-Force Attack (Exhaustive Key Search). The attacker systematically tries every possible decryption key against the intercepted ciphertext until a meaningful plaintext is obtained.

Security Principle Challenged: Confidentiality. The attacker is trying to read secret information that was meant to be kept private. If the attack succeeds, the encrypted data's secrecy is compromised.

Q10 A hospital system was hacked and patient data was modified without authorization. Which component of the CIA triad was violated? Suggest two preventive measures.

Violated Component: Integrity — Patient data was modified without authorization. This is an unauthorized alteration of information, directly violating data integrity.

Two Preventive Measures:

  1. Strict Access Control + Multi-Factor Authentication (MFA): Implement role-based access control (RBAC) ensuring only authorized medical staff can modify patient records, combined with MFA to verify identity before granting write access.
  2. Tamper-Evident Audit Logging with Integrity Checks: Maintain immutable audit logs recording every data modification (who, what, when). Use cryptographic checksums/digital signatures to detect any unauthorized changes to records.
Q11 An organization uses weak passwords for encrypted archives. Attackers successfully guess passwords using automated tools. What type of attack is this? How can it be prevented?

Attack Type: Password Guessing / Brute-Force Attack — specifically a dictionary-based automated attack. Automated tools systematically try common passwords, dictionary words, and variations.

Prevention Measures:

  1. Strong Passphrase Policy: Enforce minimum length, complexity requirements (uppercase, lowercase, numbers, symbols). Encourage passphrases over passwords.
  2. Rate Limiting & Account Lockouts: Limit login attempts; temporarily lock accounts after repeated failures.
  3. Multi-Factor Authentication (MFA): Even if the password is guessed, a second factor (OTP, biometric, hardware key) blocks unauthorized access.
  4. Key Derivation Hardening: Use algorithms like PBKDF2, bcrypt, or Argon2 to make password-based encryption computationally expensive to brute-force.
