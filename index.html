Lecture 1: Security Foundations

  • Computer security protects systems in adversarial environments against unintended use.
  • Core properties introduced: confidentiality, integrity, availability, authenticity, anonymity.
  • Focus: why systems fail and how to secure them (not offensive misuse).

Lecture 2: Security Goals and Attacks

  • CIA triad definitions and tools (encryption, access control, authentication, backups, redundancy).
  • Integrity split into data integrity and system integrity.
  • Attacks: passive (eavesdropping, traffic analysis) and active (masquerade, modification, replay, DoS).

Lecture 4: Cryptosystem Basics and Brute Force

  • Formal model: C = E(M) and M = D(C) with key secrecy and open algorithms.
  • Brute force: try keys until meaningful plaintext appears.
  • Unicity connection: for natural language, enough ciphertext can uniquely reveal the correct key.

Lecture 6: Cryptanalysis, Block Ciphers, and Modes

  • Cryptanalysis models: ciphertext-only, known-plaintext, chosen-plaintext, chosen-ciphertext.
  • Block ciphers need padding so message length is a multiple of block size.
  • Modes covered: ECB, CBC, CFB, OFB with different security and error/streaming behavior.

Playfair, Vigenere, and Unicity Distance

  • Vigenere improves over Caesar by polyalphabetic substitution using a repeating key.
  • Playfair encrypts digraphs (letter pairs), reducing simple single-letter frequency leakage.
  • Unicity distance: minimum ciphertext needed so only one key yields meaningful plaintext.

Unicity intuition: short ciphertext can match multiple keys; longer ciphertext usually collapses to one meaningful key.

Sample Questions: Solved

Part A

  1. System integrity: system runs as intended without unauthorized manipulation.
    Integrity vs confidentiality: confidentiality prevents unauthorized reading; integrity prevents unauthorized change.
    Example: If salary data is leaked, confidentiality fails. If salary numbers are altered, integrity fails.
  2. CIA in banking:
    Confidentiality: account data encrypted and access-controlled.
    Integrity: transactions protected by checks, signatures, and audit logs.
    Availability: redundant servers and DDoS protection keep services online.
  3. Cryptanalysis attacks: ciphertext-only, known-plaintext, chosen-plaintext (offline/adaptive), chosen-ciphertext.
    Ciphertext-only: attacker only sees ciphertexts and tries to recover plaintext/key.
    Chosen-plaintext: attacker chooses plaintexts and observes ciphertexts to infer key behavior.
  4. Brute force attack: exhaustive key search until meaningful decryption appears.
    Practical when keyspace is small, keys are weak/predictable, attacker has high compute, or enough ciphertext exposes a unique valid plaintext.
  5. Block cipher: encrypts fixed-size blocks using same secret key.
    Modes: ECB: independent blocks; leaks patterns. CBC: chains blocks with IV; hides patterns but sequential dependency. CFB: stream-like feedback mode; useful for streaming. OFB: keystream mode; avoids error propagation from ciphertext corruption.
    Example mode trade-off (CBC): Advantage: hides plaintext patterns. Disadvantage: packet loss/order issues hurt decryption.
  6. PKCS#5 padding: append $k$ bytes each valued $k$ where $k = b - (n \bmod b)$, with block size $b$ in bytes.
    Needed because block ciphers require plaintext length to be exact block multiple; also removes ambiguity during unpadding.
  7. Vigenere is stronger than Caesar because Caesar uses one fixed shift for all letters, while Vigenere uses key-based varying shifts (polyalphabetic), weakening simple frequency attacks.
  8. Playfair encryption: build a 5x5 key matrix (I/J combined), split plaintext into digraphs, then encrypt by row/column/rectangle rules.
    Stronger than simple substitution because it encrypts letter pairs, obscuring single-letter statistics.
  9. Scenario attack: brute-force (exhaustive key search).
    Challenged principle: confidentiality.
  10. Violated CIA component: integrity (unauthorized modification of patient data).
    Two preventive measures: strict access control + MFA, and tamper-evident audit logging with integrity checks.
  11. Attack type: password guessing / brute-force (often dictionary-based automation).
    Prevention: strong passphrase policy, rate limits/account lockouts, MFA, and key-derivation hardening (e.g., PBKDF2/Argon2) for password-based encryption.
